# PflegeProfi - Prüfungsvorbereitung für Pflegekräfte

## Projektübersicht
Digitales Lern- und Simulationssystem zur Vorbereitung ausländischer Pflegekräfte auf deutsche Anerkennungs-/Examensprüfungen. PoC fokussiert auf mündliche Prüfungssimulation.

## Technischer Stack
- **Framework**: Next.js 14 (App Router), TypeScript, Tailwind CSS
- **Datenbank**: PostgreSQL via Prisma (Abacus-hosted)
- **Auth**: NextAuth v4 mit Credentials Provider (Email/Passwort)
- **LLM**: Abacus.AI API (gpt-5.4-mini), echtes Token-Streaming
- **i18n**: i18next + react-i18next (DE/TR)

## Design-Sprache
- Professionelles, medizinisches Theme
- Primary: Teal/Blue (default primary token)
- Orange-Akzent für türkische Inhalte
- Fokus auf Lesbarkeit (viel Text in Simulationen)
- Mobile-First responsive Design
- Light theme als Standard

## Seitenstruktur
- `/login` - Anmeldeseite
- `/register` - Registrierungsseite
- `/dashboard` - Übersicht (Szenarien, absolvierte Simulationen, Stats)
- `/simulation/[templateId]/briefing` - Aufgabenstellung vor Simulation
- `/simulation/[templateId]/chat` - Chat-Interface mit LLM (echtes Streaming)
- `/evaluation/[simId]` - Bewertungsergebnis (Scores + Feedback DE/TR)
- `/glossary` - Medizinisches Glossar (20 Begriffe DE↔TR)

## Zweisprachigkeit (Kernfeature)
- UI-Sprache: DE/TR Toggle (i18next)
- LLM-Sprachmodus: "Nur Deutsch" vs. "Mit türkischer Unterstützung"
- Glossar-Tooltips: Fachbegriffe im Chat werden mit TR-Übersetzung angezeigt
- Bewertungs-Feedback: Zweisprachig (Tabs DE/TR)

## Datenbankschema
- User, SimulationTemplate, UserSimulation, SimulationInteraction, Evaluation, GlossaryTerm, HandbookSection
- SimulationTemplate hat `checklist` (JSON) für kontextabhängige Bewertungscheckliste
- UserSimulation hat `documentation` (Text) für Pflegedokumentation
- Evaluation hat `checklistResults` (JSON), `docFeedbackDe/Tr`, `docScore` für transparente Bewertung
- 5 Basis-Szenarien seeded (beginner/intermediate/advanced)
- Dynamische Szenario-Generierung via LLM (API: /api/simulation/generate)
  - 12 Themenkategorien aus Handbuch-Struktur (lib/topic-categories.ts)
  - 4 Schwierigkeitsgrade: beginner, intermediate, advanced, extreme
  - 3 Simulationstypen: oral_exam, patient_conversation, written_task
  - Neue Simulation-Seite: /simulation/new (3-Schritt-Wizard)
  - Checkliste wird kontextabhängig generiert (Fachsprache-Bewertung je nach Typ)
- Dashboard-Filter nach Schwierigkeit und Typ
- 20 Glossarbegriffe seeded
- RAG: HandbookSection Tabelle für Handbuch-Inhalte (Part 2 geladen, Rest ausstehend)

## Bewertungssystem
- Checklisten-basiert: 8-15 spezifische Items pro Simulation
- Kontextabhängig: Fachsprache im Patientengespräch = Fehler, in Dokumentation = notwendig
- Patientengespräche enthalten subtile Auffälligkeiten (Ängstlichkeit, kognitive Ausfälle)
- Dokumentationsphase: Bei patient_conversation & written_task muss User Pflegedokumentation erstellen
- Dokumentations-Score fließt mit 30% in Gesamtbewertung ein
- Checklist-Items haben Gewichtung (1=normal, 2=wichtig, 3=kritisch)

## Spracheingabe
- Web Speech API (browser-nativ, kein API-Key nötig)
- Hook: `hooks/use-speech-recognition.ts`
- Sprache wird automatisch nach UI-Sprache gesetzt (de-DE / tr-TR)
- Mikrofon-Button im Chat-Interface mit Live-Transkription
- Funktioniert in Chrome, Edge, Safari (nicht Firefox)

## API-Routen
- `/api/simulation/stream` - SSE Streaming für Chat
- `/api/simulation/evaluate` - JSON Bewertung nach Simulation
- `/api/simulation/generate` - Dynamische Szenario-Generierung via LLM
- `/api/templates`, `/api/simulations`, `/api/glossary` - CRUD
